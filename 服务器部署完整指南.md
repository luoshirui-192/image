# 服务器部署完整指南

本文档说明如何在**一台服务器**上从零部署「图像路径式数据库管理系统」，供局域网或内网用户通过浏览器访问。

**推荐方式：Docker 一键部署**（无需单独安装 Python、Node、MySQL、Nginx）。

---

## 1. 部署架构

```
用户电脑（浏览器）
        │  http://服务器IP:端口
        ▼
┌───────────────────────────────────────┐
│              服务器                      │
│  ┌─────────┐    ┌──────────┐          │
│  │  web    │───►│ backend  │          │
│  │ Nginx   │    │ Gunicorn │          │
│  │ + 前端  │    │ Django   │          │
│  └─────────┘    └────┬─────┘          │
│       :80            │                │
│                      ▼                │
│                 ┌─────────┐             │
│                 │   db    │             │
│                 │  MySQL  │             │
│                 └─────────┘             │
│  数据卷：mysql_data / upload_data       │
└───────────────────────────────────────┘
```

- 用户**只访问** Nginx 暴露的 HTTP 端口（默认 80）
- 数据库、后端 API **不应对外**直接开放
- 图片经 `/api/images/*` 鉴权访问，不直接暴露 `upload/` 目录

---

## 2. 服务器要求

| 项目 | 建议 |
|------|------|
| 操作系统 | **Linux**（Ubuntu 22.04 / Debian 12 等，生产首选） |
| | Windows Server 也可用 Docker Desktop，见下文备注 |
| CPU | 2 核及以上 |
| 内存 | **4GB 及以上**（首次构建前端镜像较耗内存） |
| 磁盘 | 20GB 及以上（视图片存储量增减） |
| 网络 | 固定内网 IP；若外网访问需公网 IP 或域名 |
| 软件 | Git、Docker Engine、Docker Compose 插件 |

---

## 3. 部署前准备

### 3.1 确认服务器 IP

```bash
# Linux 查看 IP
ip addr
# 或
hostname -I
```

记下内网 IP，例如 `192.168.1.100`，后续配置 `PUBLIC_URL` 要用。

### 3.2 安装 Git

```bash
# Ubuntu / Debian
sudo apt update
sudo apt install -y git
```

### 3.3 安装 Docker（Linux）

```bash
# 官方一键脚本（Ubuntu / Debian 等）
curl -fsSL https://get.docker.com | sudo sh

# 当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER
# 重新登录 SSH 后生效

# 验证
docker --version
docker compose version
```

### 3.4 Windows Server 备注

1. 安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/) 或 Docker Engine  
2. 确保 Docker 处于 **Running** 状态  
3. 使用 PowerShell 执行 `.\start.ps1`（与 Linux 的 `./start.sh` 等价）

## 4. 获取项目代码

### 方式 A：从 GitHub 克隆（推荐）

```bash
cd /opt
sudo git clone https://github.com/你的用户名/你的仓库名.git image_db
sudo chown -R $USER:$USER image_db
cd image_db
```

### 方式 B：上传 ZIP

1. 在 GitHub 下载 ZIP 并上传到服务器  
2. 解压到 `/opt/image_db`（路径可自定，与下文 `PROJECT_ROOT` 一致即可）

---

## 5. 配置环境变量

### 5.1 创建 `.env`

```bash
cd /opt/image_db
cp .env.docker.example .env
nano .env   # 或 vim .env
```

### 5.2 必改项（生产环境）

```env
# 用户浏览器访问的完整地址（无末尾斜杠）
PUBLIC_URL=http://192.168.1.100

# 对外 HTTP 端口（默认 80；若占用可改为 8080，PUBLIC_URL 也要带端口）
HTTP_PORT=80

# 数据库密码（容器内 MySQL，与手工建库无关）
MYSQL_ROOT_PASSWORD=请设强密码_root
MYSQL_PASSWORD=请设强密码_app

# Django 密钥（两个都要换，且彼此不同）
SECRET_KEY=请设随机长字符串
IMAGE_ACCESS_SECRET=请设另一随机长字符串
```

生成随机密钥（服务器上，需已安装 Python 3）：

```bash
python3 scripts/generate_secret_key.py
# 执行两次，分别填入 SECRET_KEY 和 IMAGE_ACCESS_SECRET
```

### 5.3 配置说明

| 变量 | 说明 |
|------|------|
| `PUBLIC_URL` | 必须与用户浏览器地址栏一致，如 `http://192.168.1.100` 或 `http://192.168.1.100:8080` |
| `HTTP_PORT` | Docker 映射到宿主机的端口 |
| `MYSQL_*` | 容器内 MySQL 账号，首次启动自动建库 |
| `SECRET_KEY` / `IMAGE_ACCESS_SECRET` | 生产环境禁止使用模板默认值 |

执行 `start.sh` 时，`docker/set-env.py` 会根据 `PUBLIC_URL` 自动写入 `ALLOWED_HOSTS` 和 `CORS_ALLOWED_ORIGINS`。

---

## 6. 启动部署

### 6.1 Linux

```bash
cd /opt/image_db
chmod +x start.sh
./start.sh
```

### 6.2 Windows Server

```powershell
cd D:\image_db   # 你的实际路径
.\start.ps1
```

### 6.3 首次启动时发生什么

1. 检查 Docker 是否可用  
2. 若无 `.env` 则从模板复制  
3. 同步 `PUBLIC_URL` → `ALLOWED_HOSTS` / `CORS`  
4. `docker compose up -d --build`  
   - 构建前端（npm）与后端（pip）镜像  
   - 启动 `db`、`backend`、`web` 三个容器  
5. MySQL 首次初始化：自动执行 `sql/` 建库与种子数据  
6. 后端执行 `migrate --fake-initial`  

**首次约 5～15 分钟**（视网络与机器性能）；之后重启通常 1 分钟内。

### 6.4 成功标志

终端输出类似：

```
==========================================
 系统已启动
 浏览器访问: http://192.168.1.100
 默认账号: admin / admin123
 停止服务: docker compose down
==========================================
```

## 7. 验收测试（在服务器上执行）

### 7.1 容器状态

```bash
docker compose ps
```

三个服务均应为 `running`：

| 服务名 | 说明 |
|--------|------|
| `db` | MySQL |
| `backend` | Django API |
| `web` | Nginx + 前端 |

### 7.2 健康检查

```bash
curl -s http://127.0.0.1/api/health/ | head
# 若 HTTP_PORT=8080，改为 http://127.0.0.1:8080/api/health/
```

响应中 `"code": 0` 表示 API 正常。

### 7.3 冒烟测试（可选，需 Python 3）

```bash
python3 scripts/smoke_test.py --base-url http://127.0.0.1
# 或带端口 / 带公网地址
python3 scripts/smoke_test.py --base-url http://192.168.1.100
```

### 7.4 浏览器验收

在服务器本机或用户电脑上打开 `PUBLIC_URL` 配置的地址，使用 `admin / admin123` 登录（**上线后请立即改密码**）。

## 8. 开放网络访问（关键）

仅服务器部署成功还不够，需保证**用户电脑能连到服务器端口**。

### 8.1 Linux 防火墙

```bash
# UFW 示例（HTTP_PORT=80）
sudo ufw allow 80/tcp
sudo ufw reload

# HTTP_PORT=8080 时
sudo ufw allow 8080/tcp
```

### 8.2 云服务器安全组

在阿里云 / 腾讯云 / AWS 等控制台，为实例**入站规则**放行 `HTTP_PORT`（TCP）。

### 8.3 用户电脑如何访问

| 场景 | 用户浏览器输入 |
|------|----------------|
| 同局域网 | `http://192.168.1.100` |
| 改了端口 8080 | `http://192.168.1.100:8080` |
| 有域名 | `http://imgdb.company.com`（`PUBLIC_URL` 同步改为该域名） |

用户电脑**不需要**安装 Docker 或任何依赖，只需浏览器。

### 8.4 连通性自检（在用户电脑上）

```powershell
# Windows PowerShell
Test-NetConnection 192.168.1.100 -Port 80
```

```bash
# Linux / macOS
curl -I http://192.168.1.100
```

## 9. 安全加固（上线必做）

- [ ] 修改 `admin` 默认密码（种子数据为 `admin123`）  
- [ ] `.env` 中 `SECRET_KEY`、`IMAGE_ACCESS_SECRET`、`MYSQL_*` 已更换  
- [ ] 仅开放必要端口（80/443 或自定义 `HTTP_PORT`）  
- [ ] 不要将 MySQL 3306、后端 8000 映射到公网  
- [ ] 定期备份（见第 11 节）

## 10. 日常运维

### 10.1 常用命令

```bash
cd /opt/image_db

docker compose ps              # 查看状态
docker compose logs -f         # 全部日志
docker compose logs backend    # 后端日志
docker compose logs web        # Nginx 日志
docker compose restart         # 重启所有服务
docker compose down            # 停止（数据保留）
docker compose up -d           # 再次启动（不重新 build）
```

### 10.2 修改配置后

若改了 `.env`（尤其是 `PUBLIC_URL`）：

```bash
./start.sh
# 或
python3 docker/set-env.py && docker compose up -d
```

若改了代码或 `docker-compose.yml`：

```bash
docker compose up -d --build
```

### 10.3 开机自启

Docker 服务设为开机启动后，容器已配置 `restart: unless-stopped`，服务器重启后会自动拉起。

```bash
# Linux 启用 Docker 开机自启
sudo systemctl enable docker
```

## 11. 备份与恢复

### 11.1 数据位置

Docker 命名卷（`docker volume ls` 可见）：

| 卷名 | 内容 |
|------|------|
| `image-db_mysql_data` | 数据库 |
| `image-db_upload_data` | 上传图片 |
| `image-db_thumb_cache` | 缩略图缓存 |

### 11.2 逻辑备份（在宿主机项目目录）

需安装 `mysqldump` 客户端，并将 `backend/.env` 指向容器数据库时较麻烦；**推荐直接对 db 容器备份**：

```bash
# MySQL 逻辑备份示例
docker compose exec db mysqldump -uimage_db -p"${MYSQL_PASSWORD}" image_db | gzip > backups/manual_$(date +%Y%m%d).sql.gz
```

`upload` 数据在卷中，可用：

```bash
docker run --rm -v image-db_upload_data:/data -v $(pwd)/backups:/backup alpine \
  tar czf /backup/upload_$(date +%Y%m%d).tar.gz -C /data .
```

### 11.3 定时备份

参考 `deploy/backup/cron.example`，将路径改为 `/opt/image_db`。

## 13. 故障排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `docker: command not found` | 未装 Docker | 见 3.3 节 |
| 构建失败 / 内存不足 | 内存 < 4GB | 增加 swap 或换更大机器 |
| 端口被占用 | 80 已被占用 | `.env` 改 `HTTP_PORT=8080`，`PUBLIC_URL` 带端口 |
| 502 Bad Gateway | 后端未就绪 | `docker compose logs backend`，等 MySQL 初始化完成 |
| 能开页不能登录 | DB 未就绪或 CORS | 等 1～2 分钟；检查 `PUBLIC_URL` 与浏览器地址一致 |
| 外网无法访问 | 防火墙/安全组 | 见第 8 节 |
| `python` 找不到（start.sh） | 未装 Python 3 | `apt install python3`，或手动 `docker compose up -d --build` 并自行改 `.env` 中 `ALLOWED_HOSTS` |

### 13.1 查看后端是否连上数据库

```bash
docker compose logs backend --tail 100
```

### 13.2 进入容器调试

```bash
docker compose exec backend bash
docker compose exec db mysql -uimage_db -p image_db
```

### 13.3 彻底重置（清空所有数据）

```bash
docker compose down -v   # 会删除卷，慎用！
./start.sh
```
