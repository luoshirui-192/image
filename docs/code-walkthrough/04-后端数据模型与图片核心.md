# 第 4 章：后端数据模型与图片核心

---

## `backend/images/models.py`（220 行）

所有业务表 **`managed = False`**：结构以 `sql/*.sql` 为准，Django 只做 ORM 映射。

### ImageCategory（行 4-20）

| 行/字段 | 说明 |
|---------|------|
| 5 | `category_name` 分类显示名 |
| 6 | `sort` 排序，越小越靠前 |
| 7 | `create_time` 创建时间 |
| 10 | `managed = False` |
| 11 | 物理表名 `image_category` |

### ImageInfo（行 23-54）— **核心表**

| 字段 | 说明 |
|------|------|
| `image_name` | 原始文件名 |
| `image_path` | **相对路径** `upload/20260703/2/uuid.jpg` |
| `image_width/height` | 像素 |
| `file_size` | 字节 |
| `file_suffix` | 扩展名 |
| `file_hash` | SHA256 去重 |
| `upload_time/update_time` | 时间 |
| `upload_user` | 上传者 |
| `is_delete` | 软删 0/1 |
| `category_id` | 关联分类（路径目录也用此 id） |
| `tags` | 逗号分隔标签 |

索引 `idx_list_category` 支持按分类列表查询。

### BlobMigrationSource（行 57-84）— 迁移配置

| 字段 | 说明 |
|------|------|
| `source_table` | 源表名 |
| `blob_column` / `blob_columns` | 单列或 JSON 多列 |
| `source_pk_column` | 源主键 |
| `category_id` | 迁入哪个分类 |
| `where_clause` | 附加过滤 |
| `db_alias` | default / legacy / external 连接别名 |

### ImageSourceMap（行 87-106）— 迁移映射

唯一键 `(source_table, source_id, source_column)` → `image_info_id`。  
**一行多 BLOB 列**各占一条映射。

### BlobTableView（行 109-134）— 「数据库模拟」保存的配置

| 字段 | 说明 |
|------|------|
| `db_alias` + `database_name` + `source_table` | 定位源对象 |
| `blob_columns` | 要展示/替换的列 |
| `where_clause` | 浏览过滤 |

### ExternalDbConnection（行 137-161）

外部 MySQL 连接信息；`password_encrypted` 加密存库。

### BlobMigrationJob / BlobMigrationJobError（行 164-219）

后台任务状态机：`pending→running→completed/failed/cancelled`；错误行写入 error 表。

## `backend/images/views.py` — 上传与分类

### CategoryListCreateView

| 方法 | 说明 |
|------|------|
| GET | 列出全部分类，按 sort |
| POST | 创建分类，写操作日志 |

### CategoryDetailView

| 方法 | 说明 |
|------|------|
| PATCH | 改名/排序 |
| DELETE | 若 `image_info` 仍有该 category_id 则拒绝 |

### ImageUploadView

| 步骤 | 说明 |
|------|------|
| 取 `category_id`、文件列表 | multipart |
| 调 `services.save_uploaded_image` | 校验、算 hash、写 storage、插 DB |
| 返回每张图 metadata | 供前端展示 |
