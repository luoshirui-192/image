## `backend/config/settings.py`（311 行，分块）

### 行 1-16：路径与 env

| 行 | 说明 |
|----|------|
| 13 | `BASE_DIR` = `backend/` 目录 |
| 14 | `PROJECT_ROOT` = 仓库根（`upload/` 在这） |
| 16 | `load_dotenv(BASE_DIR/.env)` 加载 secrets |

### 行 22-28：安全基础

| 行 | 说明 |
|----|------|
| 22 | `SECRET_KEY` JWT/签名根密钥 |
| 23 | `DEBUG` 开发 True / 生产 False |
| 24-28 | `ALLOWED_HOSTS` 逗号分隔 |

### 行 34-49：INSTALLED_APPS

注册 Django 内置 + DRF + JWT + CORS + Swagger + 四个业务 app：`users images sqlquery logs`。

### 行 51-60：MIDDLEWARE

请求链：安全 → **CORS** → Session → Common → CSRF → Auth → Message → XFrame。

### 行 85-131：数据库

| 行 | 说明 |
|----|------|
| 85-86 | 测试或 `DB_ENGINE=sqlite` 时用内存库 |
| 89-101 | `_mysql_options`：字符集、MySQL 5.1 兼容 hack |
| 104-114 | `_build_mysql_database`：从 `DB_*` 或 `LEGACY_DB_*` 读连接 |
| 127-131 | 默认库 + 可选 `legacy` 第二连接（旧库 BLOB 源） |

### 行 137：AUTH_USER_MODEL

使用自定义表 `users.SysUser`，不是 Django 默认 User。

### 行 164-200：项目路径与业务参数

| 变量 | 作用 |
|------|------|
| `UPLOAD_ROOT` | 本地文件根 |
| `THUMB_CACHE_ROOT` | 缩略图磁盘缓存 |
| `MAX_UPLOAD_SIZE_*` | 上传大小上限 |
| `STORAGE_BACKEND` | `local` 或 `minio` |
| `MINIO_*` | S3 兼容存储连接 |
| `IMAGE_ACCESS_*` | 图片 URL 临时 token |
| `SQL_*` | SQL 超时、最大行数、是否强制 WHERE |
| `BLOB_MIGRATION_*` | 迁移批大小、并发、轮询间隔 |

### 行 287-304：DRF 与 JWT

| 行 | 说明 |
|----|------|
| 288-290 | 默认 JWT 认证 |
| 291-293 | 默认需登录 |
| 295 | 统一异常格式 `utils.exceptions` |
| 300-304 | access 2h、refresh 7d（可 env 改） |

## `backend/config/views.py` — HealthView

返回 `{status: ok}` 等，供 Docker health / 首页展示。

## `backend/config/settings_views.py` — SystemConfigView

聚合返回前端 Settings 页需要的只读/可写配置项。

## `backend/config/db_backend/`

自定义 Django MySQL 引擎，兼容 **MySQL 5.1** 老库（无 utf8mb4 等）。

---

## `backend/utils/path_builder.py`（162 行，核心）

| 行 | 说明 |
|----|------|
| 1-9 | 文档：路径格式 `upload/{YYYYMMDD}/{category_id}/{uuid}.ext` |
| 19-27 | 上海时区、路径正则 |
| 30-39 | `ParsedUploadPath` 数据类 |
| 42-47 | 后缀规范化（jpeg→jpg） |
| 50-60 | 日期目录名 YYYYMMDD |
| 63-93 | **`build_relative_path()`**：生成写入 DB 的相对路径 |
| 96-118 | `resolve_upload_file()`：相对路径 → 绝对路径（防目录穿越） |
| 133-136 | `is_valid_relative_path()` 校验格式 |
| 138-151 | `parse_relative_path()` 反向解析 |

**每一张上传/迁移的图片**路径都由此生成。

---

## `backend/utils/storage.py`（273 行，分块）

| 函数/类 | 作用 |
|---------|------|
| `get_storage_backend()` | 按 settings 返回 Local 或 MinIO 实现 |
| `LocalStorage` | 读写 `UPLOAD_ROOT` 下文件 |
| `MinioStorage` | boto3/minio 客户端读写 bucket+prefix |
| `save_bytes` / `read_bytes` / `exists` | 统一抽象 |

迁移与上传**只调 storage 抽象**，不关心底层是磁盘还是 MinIO。

---

## `backend/utils/file_security.py`

| 职责 | 说明 |
|------|------|
| 魔数检测 | 防止扩展名伪造 |
| 尺寸限制 | 与 settings 一致 |
| 图片解码 | Pillow 验证是否真图片 |

---

## `backend/utils/sql_validator.py`

白名单/黑名单：禁止 `DROP`、`UPDATE` 无 WHERE 等（SQL 面板安全）。

---

## `backend/utils/responses.py`

```python
success_response(data, message, status)
error_response(message, code, status)
```

全 API 统一 `{code, message, data}` JSON 形状；前端 `request.js` 据此解析。

---

## `backend/utils/exceptions.py`

DRF `custom_exception_handler`：把 Django/DRF 异常转成上面格式。

---

## `backend/utils/audit.py`

`write_operate_log(request, action, detail)` — 写 `operate_log` 表。
