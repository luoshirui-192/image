## `backend/images/management/commands/`

Django 自定义命令：`python manage.py <命令名>`

| 命令文件 | 命令名 | 每一行的业务含义 |
|----------|--------|------------------|
| `process_blob_migration_jobs.py` | `process_blob_migration_jobs` | **scheduler 容器核心**：循环取 pending job，调 `execute_migration_job_batches` |
| `migrate_blob_images.py` | `migrate_blob_images` | CLI 手动迁移；参数 `--source-id` 或 `--source-table --blob-column --category-id` |
| `run_scheduled_maintenance.py` | `run_scheduled_maintenance` | 打包：清理软删图、清日志、清 thumb cache |
| `cleanup_deleted_images.py` | `cleanup_deleted_images` | 物理删除超过保留期的软删文件 |
| `dedupe_images.py` | `dedupe_images` | 按 file_hash 去重 |
| `warm_thumb_cache.py` | `warm_thumb_cache` | 批量生成缩略图 |
| `cleanup_thumb_cache.py` | `cleanup_thumb_cache` | 删除过期缓存文件 |
| `backfill_file_hash.py` | `backfill_file_hash` | 老数据补 hash 列 |

### `process_blob_migration_jobs.py` 典型结构（逐块）

1. `add_arguments` — 可选 `--once` 只跑一轮  
2. `handle` — while True: 查 pending jobs → 标记 running → 调 service → sleep poll_seconds

## `docker/maintenance-loop.sh`

| 行块 | 说明 |
|------|------|
| 环境变量 | `MAINTENANCE_INTERVAL_HOURS`、`BLOB_MIGRATION_POLL_SECONDS` |
| 循环 | sleep → `process_blob_migration_jobs --once` → `run_scheduled_maintenance` |

## `docker/backend-entrypoint.sh`

| 步骤 | 说明 |
|------|------|
| wait-for mysql | 直到 db 健康 |
| migrate | Django migrations |
| ensure triggers / file_hash | 补丁脚本 |
| exec gunicorn | 启动 WSGI |

## 测试文件（阅读顺序）

| 文件 | 覆盖 |
|------|------|
| `tests.py` | 上传、分类 |
| `tests_serve.py` | 图片访问 token |
| `tests_crud.py` | 软删恢复 |
| `tests_blob_migration.py` | 迁移逻辑、多列 |
| `tests_blob_table_view.py` | 浏览 rows、path 替换 |
| `tests_blob_catalog.py` | 目录 API |
| `tests_external_db.py` | 外部连接 |

**读测试 = 读规格：** 每个 `def test_*` 描述一个期望行为。

运行：`python manage.py test images`（sqlite 内存库）。
