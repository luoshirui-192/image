## `docker-compose.app.external-db.example.yml` → `docker-compose.app.override.yml`

宿主机已跑 MySQL（如 `ara_fp_analyst`）时使用：

| 配置 | 说明 |
|------|------|
| `db.profiles: [with-local-mysql]` | 默认**不**启动 Docker 内 MySQL，避免占用 `127.0.0.1:3306` |
| `backend/scheduler.environment.DB_HOST` | `host.docker.internal` 指向宿主机 |
| `DB_PASSWORD` | 写在 `.env`，对应应用库用户（如 `fp_user`） |

**启动：**

```bash
docker compose -f docker-compose.app.yml -f docker-compose.app.override.yml up -d --build
```

`./start-app.sh` **不会**自动加载 override；宿主机 MySQL 场景请用上面命令。

## `start-app.sh` / `start-app.ps1`

| 步骤 | 作用 |
|------|------|
| 检查 Docker | 未安装则退出 |
| 复制 `.env.app.example` → `.env`（若不存在） | 提供默认环境 |
| `docker compose -f docker-compose.app.yml up -d --build` | 构建并后台启动 |

Windows 版 `start-app.ps1` 逻辑等价，语法为 PowerShell。

## 环境变量模板

### `.env.app.example`

Machine A 必填项典型包括：`MYSQL_ROOT_PASSWORD`、`MYSQL_PASSWORD`、`SECRET_KEY`、`MINIO_ENDPOINT`、密钥等。  
**每一行**都是 `KEY=值`：`KEY` 对应 `settings.py` 里 `os.getenv("KEY")` 的读取名。

### `backend/.env.example`

本地直接跑 `manage.py runserver` 时用；`DB_HOST=127.0.0.1` 等。

### `frontend/.env.production.example`

| 行 | 说明 |
|----|------|
| `VITE_API_BASE_URL=/api` | 生产构建时 axios 的 API 前缀（Nginx 同域反代） |

## 如何对照

1. 打开 `docker-compose.app.yml`，从第 7 行锚点开始，理解 backend 环境。
2. 打开 `backend/config/settings.py` 第 164-200 行，看同名环境变量如何被消费。
3. 部署时在 Machine A 执行 `start-app.sh`，对照 README-MACHINE-A 验证 health。
